通过取模转换

private static int getUnsignedByte(byte b) {

}

// unsigned byte -> int
private static int getUnsignedByte(byte b) {
    return b & 0x0FF;
}

带符号整数溢出的应用

一般来说，如果出现整数溢出的情况，我们就不能期待计算结果是正确的，然而在数据范围远远大于整数可以表示的范围，且又有某些限制条件的场景下，或许可以应用整数溢出的特性来保证最终结果的正确性。

两数的差值

8位带符号整数 要求计算 a 和 b 两数（a < b）的差值（距离）c ，题目表示经过一系列的计算，最终 a b 可能会远大于 8位带符号整数 的表示范围，但是保证他们的差值 c 在 8位带符号整数 的表示范围内（ ）。

在这样的场景下，a b 两数都可能会溢出，但是他们的差值 却能够正常得到计算，因为无论 a b 在环上哪个位置，他们在环上a 到 b 的顺时针方向圆弧线段 c 始终保证在 8位带符号整数 的表示范围内。
