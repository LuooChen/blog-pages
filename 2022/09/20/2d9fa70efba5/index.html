哈希函数

发布于 2022-09-20 算法  hash 

概述

哈希函数（英语：Hash function）又称散列算法、散列函数，是一种从任何一种数据中创建小的数字“指纹”的方法。哈希函数把消息或数据压缩成摘要，使得数据量变小，将数据的格式固定下来。

哈希函数（散列函数）能够将任意长度的输入值转变成固定范围的值输出，该值称为散列值，输出值通常为字母与数字组合。（即将将一个大区间的定义域，映射到一个小区间的值域上。）

如今，散列算法也被用来加密存在数据库中的密码（password）字符串，由于散列算法所计算出来的哈希值（Hash Value）具有不可逆（无法逆向演算回原本的数值）的性质，因此可有效的保护密码。

性质

基本特性

  • 确定性：如果两个哈希值是不相同的（根据同一函数），那么这两个哈希值的原始输入也是不相同的。这个特性是散列函数具有确定性的结果，具有这种性质的散列函数称为单向散列函数。
  • 散列碰撞：散列函数的输入和输出不是唯一对应关系的，如果两个散列值相同，两个输入值很可能是相同的，但也可能不同，这种情况称为“散列碰撞（collision）”（一般情况下我们会尽最大可能的避免散列碰撞）

一般属性

除了以上两个基本特性外，一般优秀的散列函数都会尽量符合以下属性：

  1. 均匀性：散列函数计算时充分利用输入的整体信息，产生的映射应当保持均匀。即不要使得映射结果集中在小区间值域的某一块区域。
  2. 高效性：散列函数的性能应该尽可能好。
  3. 强混淆特性：输入一些数据计算出散列值，然后部分改变输入值，一个具有强混淆特性的散列函数会产生一个完全不同的散列值。
  4. 非常大的定义域（接受长度非常长的输入值）：典型的散列函数都有非常大的定义域，比如SHA-2最高接受 长度的字节字符串。此处需要理解的是散列函数拥有较为庞大的输入定义域。
  5. 有限的值域（固定范围的输出值）：散列函数一定有着有限的值域，比如输出固定长度的比特串或者固定范围的正整数（哈希表常见的散列函数实现）。
  6. 不可逆性：在密码学中，散列函数必须具有不可逆性。已知散列函数 与输入 的散列值 ，无法通过 反向求出 。（这里的不可逆是指现有的计算机上无法有效计算出来，正向计算容易，无法在当前的计算机计算能力条件下逆向计算。）
  7. 抗碰撞性：对于散列函数 的输入 无法找到另一个输入 使得他们的散列值 与 相等。但是由于散列函数实际上代表着一种映射（对应关系），将一个大区间映射到一个小区间上，它一定是有冲突的，这里的无冲突同样是指已知输入 的条件下，无法通过计算找到另一个输入 使他们的散列值相同。（列如，2004年，证实MD5算法无法防止碰撞攻击，因此不适用于安全性认证，如SSL公开密钥认证或是数字签名等用途。）

散列值计算

正整数

将整数散列最常用方法是除留余数法。

我们选择散列值的范围为 ，对于任意输入正整数 ，计算 除以 的余数（）。这个函数的计算非常容易并能够有效地将散列值散布在 到 的范围内。如果 不是素数，我们可能无法利用输入整数中包含的所有信息，这可能导致我们无法均匀地散列散列值。

例如，如果输入整数 是十进制数，而 为 ，那么我们计算时只能利用到 的后 位，这可能会产生一些问题。举个简单的例子，假设输入 为美国电话号码的区号（三位区号，比如 或 ）且 。由于历史原因，美国的大部分区号中间位都是 或者 ，因此这种方法会将大量的区号 散列为小于 的索引，但如果使用素数 ，散列值的分布显然会更好（一个离 更远的素数会更好），如下图所示。与之类似，互联网中使用的 IP 地址也不是随机的，所以如果我们想用除留余数法将其散列就需要用素数（特别地，这不是2的幂）大小的数组。

除留取余法

private static final int M = 97;
private int hash(int k){
    return k % M;
}

浮点数

如果输入 是 到 之间的实数，我们可以将它乘以 并四舍五入得到一个 至 之间的索引值。尽管这个方法很容易理解，但它是有缺陷的，因为这种情况下输入浮点数 的高位起的作用更大，最低位对散列的结果没有影响。修正这个问题的办法是将输入表示为二进制数然后再使用除留余数法（ Java 就是这么做的）。

将浮点数表示为32位二进制

/**
 * Java 中 double 数据类型获取 hashcode 的方法是将其表示 64 位的二进制，
 * 然后对高位32位和低位32位进行异或后返回32位二进制，因此实际应用时就是使用二进制数进行除留余数法。
 */
public static int hashCode(double value) {
    long bits = doubleToLongBits(value);
    return (int)(bits ^ (bits >>> 32));
}

字符串

除留余数法也可以处理较长的数据， 例如字符串，我们只需将它们当作大整数即可。例如，以下的代码就能够用除留余数法计算 String s 的散列值：

private static final int R = 31;
private static final int M = 97;
private int hash(String s) {
    int hash = 0;
    for (int i = 0; i < s.length(); i++)
    hash = (R * hash + s.charAt(i)) % M;
    return hash;
}

Java 的 charAt() 函数能够返回一个 char 值，即一个非负 位整数。如果 比任何字符的值都大，这种计算相当于将字符串当作一个 位的 进制值，将它除以 并取余。一种叫 Horner 方法的经典算法用 次乘法、加法和取余来计算一个字符串的散列值。只要 足够小，不造成溢出，那么结果就能够如我们所愿，落在 至 之内。使用一个较小的素数，例如 ，可以保证字符串中的所有字符都能发挥作用。 Java 的 String 的默认实现使用了一个类似的方法。

组合类型对象

如果输入的类型含有多个整型变量，我们可以和 String 类型一样将它们混合起来。例如，假设输入的类型是 Date ，其中含有几个整型的域： day（两个数字表示的日）， month （两个数字表示的月）和 year（ 4 个数字表示的年）。我们可以这样计算它的散列值：

private static final int R = 31;
private static final int M = 97;
private int hash(int day, int month, int year) {
    int hash = (((day * R + month) % M ) * R + year) % M;
    return hash;
}

只要 足够小不造成溢出，也可以得到一个 至 之间的散列值。在这种情况下我们可以通过选择一个适当的 ，比如 ，来省去括号内的 % M 计算。和字符串的散列算法一样，这个方法也能处理有任意多整型变量的类型。

生成数组的索引

我们在实现中会将默认的 hashCode() 方法和除留余数法结合起来产生一个 到 的整数（数组长度为 ）。

private int hash(Key x){
    return (x.hashCode() & 0x7fffffff) % M;
}
