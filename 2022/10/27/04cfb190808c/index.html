单调栈

发布于 2022-10-27 CS  数据结构 

概念

单调栈是一种和单调队列类似的数据结构。单调队列主要用于 解决滑动窗口问题，单调栈则主要用于 解决NGE问题（Next Greater Element），也就是，对序列中每个元素，找到下一个比它大的元素。（当然，“下一个”可以换成“上一个”，“比它大”也可以换成“比他小”，原理不变。）

这比单调队列还简单一点。我们维护一个栈，表示“待确定NGE的元素”，然后遍历序列。当我们碰上一个新元素，我们知道，越靠近栈顶的元素离新元素位置越近。所以不断比较新元素与栈顶，如果新元素比栈顶大，则可断定新元素就是栈顶的NGE，于是弹出栈顶并继续比较。直到新元素不比栈顶大，再将新元素压入栈。显然，这样形成的栈是单调递减的。

单调栈插入流程

将一个元素插入单调栈时，为了维护栈的单调性，需要在保证将该元素插入到栈顶后整个栈满足单调性的前提下弹出最少的元素。

代码

用 Java 代码描述如下：

class MonotonicStackDemo {
    public void insert(int[] nums) {
        int n = nums.length;
        Deque<Integer> stack = new ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            int curNum = nums[i];
            while (!stack.isEmpty() && stack.peek() < curNum) {
                stack.pop();
            }
            stack.push(curNum);
        }
    }

    public static void main(String[] args) {
        MonotonicStackDemo monotonicStack = new MonotonicStackDemo();
        int[] nums = {81,45,11,0,14};
        monotonicStack.insert(nums);
    }
}

496. 下一个更大元素 I

leetcode-496. 下一个更大元素 I

题目描述

nums1 中数字 x 的 下一个更大元素 是指 x 在 nums2 中对应位置 右侧 的 第一个 比 x 大的元素。

给你两个 没有重复元素 的数组 nums1 和 nums2 ，下标从 开始计数，其中 nums1 是 nums2 的子集。

对于每个 0 <= i < nums1.length ，找出满足 nums1[i] == nums2[j] 的下标 j ，并且在 nums2 确定 nums2[j] 的 下一个更大元素 。如果不存在下一个更大元素，那么本次查询的答案是 -1 。

返回一个长度为 nums1.length 的数组 ans 作为答案，满足 ans[i] 是如上所述的 下一个更大元素 。

示例 1：

输入：nums1 = [4,1,2], nums2 = [1,3,4,2].
输出：[-1,3,-1]
解释：nums1 中每个值的下一个更大元素如下所述：
- 4 ，用加粗斜体标识，nums2 = [1,3,4,2]。不存在下一个更大元素，所以答案是 -1 。
- 1 ，用加粗斜体标识，nums2 = [1,3,4,2]。下一个更大元素是 3 。
- 2 ，用加粗斜体标识，nums2 = [1,3,4,2]。不存在下一个更大元素，所以答案是 -1 。

解决方案

我们可以使用单调栈来预处理计算 nums2 中每个元素右边的第一个更大的值，代码如下。

class Solution {
    private final static int INF = 10001;
    public int[] nextGreaterElement(int[] nums1, int[] nums2) {
        int[] nextGreater = new int[INF];
        // monotone increasing
        Deque<Integer> monotonicStack = new ArrayDeque<>();
        for (int num : nums2) {
            while (!monotonicStack.isEmpty() && num > monotonicStack.peek()) {
                nextGreater[monotonicStack.pop()] = num;
            }
            monotonicStack.push(num);
        }
        while (!monotonicStack.isEmpty()) {
            nextGreater[monotonicStack.pop()] = -1;
        }
        int n = nums1.length;
        int[] ans = new int[n];
        for (int i = 0; i < n; i++) {
            ans[i] = nextGreater[nums1[i]];
        }
        return ans;
    }
}

代码模板

在洛谷实际提交会报 MLE 应该是由于用了 Deque 数据结构导致内存超出限制，但是代码本身是正确的

import java.util.ArrayDeque;
import java.util.Deque;
import java.util.Scanner;
class Main {
    public int[] getAns(int[] nums) {
        int n = nums.length;
        Deque<Integer> stack = new ArrayDeque<>();
        for (int i = 0; i < n; i++) {
            int curNum = nums[i];
            while (!stack.isEmpty() && nums[stack.peek()] < curNum) {
                nums[stack.pop()] = i + 1;
            }
            stack.push(i);
        }
        while (!stack.isEmpty()) {
            nums[stack.pop()] = 0;
        }
        return nums;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        int[] nums = new int[n];
        for (int i = 0; i < n; i++) {
            nums[i] = sc.nextInt();
        }
        Main monotonicStack = new Main();
        int[] ans = monotonicStack.getAns(nums);
        for (int num : ans) {
            System.out.printf("%d ", num);
        }
    }
}
