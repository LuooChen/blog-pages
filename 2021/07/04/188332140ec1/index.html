位运算

发布于 2021-07-04 CS  位运算 

概念

位运算是把数字用二进制表示之后，对每一位上0或者1的运算。二进制及其位运算是现代计算机学科的基石，很多底层的技术都离不开位运算，因此位运算相关的题目也经常出现在面试中。

原码、反码、补码

以下均演示 8-bit 进行二进制展示

机器数

一个数在计算机中的二进制表示形式叫做这个数的机器数，机器数是带符号的，计算机用机器数的最高位放符号位，正数为 0，负数为 1。因为第一位是符号位，所以机器数的形式值就不等于真正的数值，将带符号的机器数对应的真正数值称为机器数的真值。原码、反码、补码是机器存储一个具体数字的编码方式，就是为了理解计算机使用二进制进行计算的原理。

例如： \[ 3 = (0000 0011)_2 \] \[ -3 = (1000 0011)_2 \]

原码

原码是机器数中最简单的一种形式，原码由符号位和数值位两部分构成。数值位部分均保持原来的样子，不需改变。如果是正数，最前面的 0 视为符号位；如果是负数，最前面的 1 视为符号位，数值位就是真值的绝对值，原码又称带符号的绝对值。原码进行加减运算时，一个正数和一个负数运算时的结果可能为正也可能为负，而且是绝对值进行比较，这样相对很不方便，因此引入了补码。

例如： \[ 3 = (0000 0011)_2 \]

\[ -3 = (1000 0011)_2 \]

反码

反码通常是用来由原码求补码或者由补码求原码的过渡码，反码跟原码是正数时一样；负数时，反码就是原码符号位除外，其他位按位取反。

例如： \[ 3 = (0000 0011)_2 \]

\[ -3 = (1111 1100)_2 \]

补码

补码是计算机把减法运算转化为加法运算的关键编码，正数的补码是它本身，负数的补码是除符号位，原码取反加 1。

例如： \[ 3 = (0000 0011)_2 \]

\[ -3 = (1111 1101)_2 \]

特殊的两个数字

在补码中，有符号整数的场景下， \(0 = (0000 0000)_2\) 以及 \(-128 = (1000 0000)_2\) 是比较特殊的两个数字。

为了保证不同时出现两种 \(0\) ( \(+0\) \(-0\) ) ，于是将 \((1000 0000)_2\) 规定为 \(-128\) 的表示。于是整数的表示取值范围就成了 \(-128\) ~ \(127\) (即\((1000 0000)_2\) ~ \((0111 1111)_2\)) 。

其中 \(0\) 和 \(-128\) 的原码和补码都是它本身。

带符号整数溢出问题

以下均演示 8位带符号整数

8位带符号整数 （例如 Java 中的 byte 数据类型），能够表示的整数范围为 \([-128, 127]\) 。按正负来分段，大致可以分为两段，即 \([-128, -1]\) \([0, 127]\) 。

也就是说，最小不能小于 \(-128\) 而最大不能大于 \(127\) 否则就会出现整数溢出的问题。比如 \(-128 - 1 = 127\) 或者 \(127 + 1 = -128\) 这样的现象。也就是说，计算结果与我们期待的不符。

带符号整数溢出形成了一个环

实际上我们仔细观察 \([-128, -1]\) \([0, 127]\) 的数据范围，整数最大值 \(127\) 加上 \(1\) 就变成了负数最小值 \(-128\) ，而且我们继续往上加，最终 \(127 + 256 = 127\) 数值又变成了 \(127\) （任何数加上 256 就变回了该数本身）。于是我们发现在 \([-128, -1]\) \([0, 127]\) 范围内，我们 \(+1\) 就相当于向右移动一步（到达最右侧后返回最左侧）， \(-1\) 就相当于向左移动一步（到达最左侧后返回最右侧）。这就形成了一个环：

\(+\) 就是顺时针转动， \(-\) 就是逆时针转动。（可以类比时钟，也可以类比MOD取余操作）

实际上，带符号整数在某种意义上可以转换成无符号整数（我们知道 Java 中没有无符号整数数据类型），即 \([0, 127]\) \([-128, -1]\) -> \([0, 255]\) ，此时 \(-128\) 表示 \(128\) ，\(-127\) 表示 \(129\) ，\(-1\) 表示 \(255\) 。

因此即便一个变量的数值溢出变成了负数，只要它表示的范围还在 \([0, 255]\) 之内。我们通过一个转换函数，使用更大位数的带符号整数类型（比如 int 32位带符号整数 ）就可以将超过 8位带符号整数 正整数上限 \(127\) 的部分表示出来（即 \([128, 255]\) ）。

例如以下的无符号转换方法：

通过取模转换

// unsigned byte -> int
private static int getUnsignedByte(byte b) {
    return (b + 256) % 256;
}

通过位运算转换

// unsigned byte -> int
private static int getUnsignedByte(byte b) {
    return b & 0x0FF;
}

这可以类推到任何位数的整数，比如 int （32-bit 整数）的范围就是 \([-2^{31}, -1]\) \([0, 2^{31} - 1]\) 。

带符号整数溢出的应用

一般来说，如果出现整数溢出的情况，我们就不能期待计算结果是正确的，然而在数据范围远远大于整数可以表示的范围，且又有某些限制条件的场景下，或许可以应用整数溢出的特性来保证最终结果的正确性。

两数的差值

8位带符号整数 要求计算 a 和 b 两数（a < b）的差值（距离）c ，题目表示经过一系列的计算，最终 a b 可能会远大于 8位带符号整数 的表示范围，但是保证他们的差值 c 在 8位带符号整数 的表示范围内（ \(0 <= c <= 127\)）。

在这样的场景下，a b 两数都可能会溢出，但是他们的差值 \(c = b - a\) 却能够正常得到计算，因为无论 a b 在环上哪个位置，他们在环上a 到 b 的顺时针方向圆弧线段 c 始终保证在 8-bit 整数的表示范围内。

（可以把它想象成 a 向右移动了 c 步到 b 的位置上，也就是说 \(a + c = b\) 因此 \(c = b - a\) ）

可以看到，实际上只要保证 a 与b 距离 c 不超过该环的 最大周长255 ，无论 a b 在哪个位置，代表环上a 到 b 的顺时针方向圆弧线段的 \(c = b - a\) 总能得到“正确的结果”（如果 c 超过 \(127\) 则会溢出成为负数，虽然此时 c 保存的结果仍然“正确”，却不能被正确的表示；但此时的 c 只要使用更大的整数类型，例如换成 Java 中的 int 32位带符号整数 进行取模运算 int ans = (c + 256) % 256 ，此时8位带符号整数 c 通过 32位带符号整数 ans 表示出了正确的结果）。因此只要限制了 c 的取值范围在 \([0, 127]\) 之内，8位带符号整数 c 都能被正确的表示。

位运算符

位运算使用补码进行运算，符号位也参与运算。

取反（NOT）

取反是一元运算符，对一个二进制数的每一位执行逻辑反操作。使数字1成为0，0成为1。

在类C语言中，取反操作符是 ~ 。

例如：

NOT 0111（十进制7）
  = 1000（十进制8）

按位或（OR）

按位或处理两个长度相同的二进制数，两个相应的二进位中只要有一个为1，该位的结果值为1。

在类C语言中，按位或操作符是 | 。

例如：

    0101（十进制5）
OR  0011（十进制3）
 =  0111（十进制7）

按位异或（XOR）

按位异或运算，对等长二进制模式或二进制数的每一位执行逻辑异或操作。操作的结果是如果某位不同则该位为1，否则该位为0。

在类C语言中，按位异或运算符是 ^ 。

例如：

     0101（十进制5）
XOR  0011（十进制3）
  =  0110（十进制6）

按位与（AND）

按位与处理两个长度相同的二进制数，两个相应的二进位都为1，该位的结果值才为1，否则为0。

在类C语言中，按位异或运算符是 & 。

例如：

     0101（十进制5）
AND  0011（十进制3）
  =  0001（十进制1）

移位

移位是一个二元运算符，用来将一个二进制数中的每一位全部都向一个方向移动指定位，溢出的部分将被舍弃，而空缺的部分填入一定的值。在类C语言中，左移使用两个小于符号 << 表示，右移使用两个大于符号 >> 表示。

如果是无符号数，做的是 逻辑移位 。

如果是有符号数，做的是 算术移位 。

算术移位

对于算术移位，左移时，把目的操作数的低位向高位移，空出的低位补0；右移时，把目的操作数的高位向低位移，空出的高位用最高位(符号位)填补。

在Java语言中，逻辑左移用 << 操作符表示，逻辑右移用 >> 操作符表示。

例如（符号数，第一位表示符号）：

注意：以下二进制数为补码形式

  • 左移3位
    1111（十进制 -1）
<<<    3（左移3位）
  = 1000（十进制 -8）
  • 右移2位
   1000（十进制 -8）
>>>   2（右移2位）
  = 1110（十进制 -2）

逻辑移位

对于逻辑移位，就是不考虑符号位，移位的结果只是数据所有的位数进行移位。根据移位操作的目的，左移时，低位补0，右移时，高位补0。

在Java语言中，逻辑左移用<<< 操作符表示，逻辑右移用 >>> 操作符表示。

例如（无符号数）：

  • 左移3位
    0001（十进制1）
<<<    3（左移3位）
  = 1000（十进制8）
  • 右移2位
    1010（十进制10）
>>>    2（右移2位）
  = 0010（十进制2）

常见位操作

注意：以下方法，均使用Java语言编写

获取数位

该方法将 1 左移 i 位，得到形如 $ (00 010 000)_2 $ 的值。接着，对这个值与 num 执行“位与”操作（ AND），从而将 i 位之外的所有位清零。最后，检查该结果是否为 0。不为 0 说明 i 位为 1，否则， i 位为 0。

boolean getBit(int num, int i) {
    return ((num & (1 << i)) != 0);
}

设置数位

setBit 先将 1 左移 i 位，得到形如 $ (00 010 000)_2 $ 的值。接着，对这个值和 num 执行“位或”操作（ OR），这样只会改变 i 位的数值。该掩码 i 位除外的位均为 0，故而不会影响 num 的其余位。

int setBit(int num, int i) {
    return num | (1 << i);
}

清零数位

该方法与 setBit 刚好相反。首先，将数字 $ (00 010 000)_2 $ 取反进而得到类似于 $ (1110 1111)_2 $ 的数字。接着，对该数字和 num 执行“位与”操作（ AND）。这样只会清零 num 的第 i 位，其余位则保持不变。

int clearBit(int num, int i) {
    int mask = ~(1 << i);
    return num & mask;
}

如果要清零最高位至第 i 位所有的数位（包括最高位和第 i 位），需要创建一个第 i 位为 1（ 1 << i）的掩码。然后，将其减 1 并得到一串第一部分全为 0，第二部分全为 1 的数字。之后我们将目标数字与该掩码执行“位与”操作（ AND），即得到只保留了最后 i 位的数字。

int clearBitsMSBthroughI(int num, int i) {
    int mask = (1 << i) - 1;
    return num & mask;
}

如果要清零第 i 位至第 0 位的所有的数位（包括第 i 位和第 0 位），使用一串 1 构成的数字（即-1）并将其左移 i + 1 位，如此便得到一串第一部分全为 1，第二部分全为 0 的数字。

int clearBitsIthrough0(int num, int i) {
    int mask = (-1 << (i + 1));
    return num & mask;
}

更新数位

将第 i 位的值设置为 v，首先，用诸如 $ (1110 1111)_2 $ 的掩码将 num 的第 i 位清零。然后，将待写入值 v 左移 i 位，得到一个 i 位为 v 但其余位都为 0 的数。最后，对之前取得的两个结果执行“位或”操作， v 为 1 则将 num 的 i 位更新为 1，否则该位仍为 0。

int updateBit(int num, int i, boolean bitIs1) {
    int value = bitIs1 ? 1 : 0;
    int mask = ~(1 << i);
    return (num & mask) | (value << i);
}
