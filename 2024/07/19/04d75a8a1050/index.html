最短路

最短路问题分为两类：单源最短路和多源最短路。前者只需要求一个固定的起点到各个顶点的最短路径，后者则要求得出任意两个顶点之间的最短路径。

Floyd-Warshall 算法

Floyd-Warshall 算法属于多源最短路算法；是用来求图中的任意两个顶点之间的最短路的。

该算法适用于任何图，不管有向无向，边权正负，但是最短路必须存在。（不能有个负环）

同时可以用于检测负环的存在。（详细请看cp-algorithms-finding-negative-cycle-in-graph）

实现

以下代码假设 阶段初始化已经完成，我们通过滚动数组的方式，实现了 的计算。（代码实现中的所有顶点为 ）

public static void floydRollArray(int[][] d) {
    int n = d.length;
    for (int k = 0; k < n; k++) {
        int[][] newd = new int[n][n];
        for (int i = 0; i < n; i++) System.arraycopy(d[i], 0, newd[i], 0, n);;
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                d[i][j] = Math.min(d[i][j], newd[i][k] + newd[k][j]);
            }
        }
    }
}

public static void floyd(int[][] d) {
    int n = d.length;
    for (int k = 0; k < n; k++) {
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                d[i][j] = Math.min(d[i][j], d[i][k] + d[k][j]);
            }
        }
    }
}

public static void floydNegativeWeight(int[][] d) {
    int n = d.length;
    for (int k = 0; k < n; k++) {
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                if (d[i][k] < INF && d[j][k] < INF) {
                    d[i][j] = Math.min(d[i][j], d[i][k] + d[k][j]);
                }
            }
        }
    }
}

Dijkstra 算法

算法，是一种求解 非负权图（没有负权边） 上单源最短路的算法。

该算法用于寻找从起始顶点 到所有其他顶点的最短路径的长度，并维护最短路路径信息。

算法描述

算法基于一种贪心的思想。

设图 （有向图或无向图）的有 个顶点和 条边，无负权边；同时指定的起始顶点为 ，现在我们想要找到起始顶点 到所有其他顶点的最短路径的长度，并输出最短路径本身。

  • 每次迭代会从未标记的所有顶点中，选出 最小的顶点 。（第一次将会选中起始顶点 ，此时 最）

  • 然后标记选中的顶点 （即设置 ）；接着对顶点 执行松弛操作。

    顶点 的松弛操作：放松从顶点 指出的所有边 ；放松边 的是指检查从起始顶点 到 的最短路径是否可以变更为，先从 到 再接边 ，如果是则说明我们找到了一条更短的路径，我们可以更新 的最短路长度，即松弛操作代码是： 于是当前迭代完成。

当完成 次迭代后，我们找到了起始顶点 到所有顶点 的最短路径长度 ，算法执行结束。

注意，每次标记顶点 时，这也表示当前顶点 已经找到了最短路径，因为我们是从所有的未标记中选中了一个最小值 ，即再接下来的迭代中，已经不会出现更小的松弛操作来更新 。

实现

暴力实现

public class Dijkstra {
    private static final int INF = 0x3fffffff;
    private int n;
    private List<int[]>[] graph;
    private int[] d;
    private int[] p;

    public Dijkstra(int n, List<int[]>[] graph) {
        this.n = n;
        this.graph = graph;
        this.d = new int[n];
        this.p = new int[n];
    }
    public void dijkstraBruteforce(int s) {
        Arrays.fill(d, INF);
        Arrays.fill(p, -1);
        boolean[] visited = new boolean[n];
        d[s] = 0;
        // n time iterations
        for (int i = 0; i < n; i++) {
            // chosen vertex u
            int u = -1;
            for (int j = 0; j < n; j++) {
                if (visited[j]) continue;
                if (u == -1 || d[j] < d[u]) {
                    u = j;
                }
            }
            if (d[u] == INF) break;
            visited[u] = true;
            // relaxations vertex u
            List<int[]> adjs = graph[u];
            for (int[] adj : adjs) {
                int v = adj[0];
                int weight = adj[1]; // len of (u -> v)
                // relaxation edge u -> v
                if (d[v] > d[u] + weight) {
                    // successful relaxation
                    d[v] = d[u] + weight;
                    p[v] = u;
                }
            }
        }
    }
}

优先队列实现

我们可以用一个优先队列来维护所有未标记的顶点 （元素 {u, d[u]} ），并在每次成功放松边 后将顶点 （元素 {v, d[v]} ）加入优先队列中，因此每次迭代可以用 的复杂度从优先队列的所有元素中，快速选出 最小的顶点 ；这样可以实现在 的时间内跑完最短路。（如果使用支持 change(key, weight) 操作（修改元素权值，重新排序）的优先队列，则可以达到 的时间复杂度）

前继节点数组 的维护也与暴力实现一样，每一次放松边 成功时，我们同时更新 。


public class Dijkstra {
    private static final int INF = 0x3fffffff;
    private int n;
    private List<int[]>[] graph;
    private int[] d;
    private int[] p;

    public Dijkstra(int n, List<int[]>[] graph) {
        this.n = n;
        this.graph = graph;
        this.d = new int[n];
        this.p = new int[n];
    }

    public void dijkstra(int s) {
        Arrays.fill(d, INF);
        Arrays.fill(p, -1);
        // {vertex, pathLen}
        PriorityQueue<int[]> minPq = new PriorityQueue<>((o1, o2) -> o1[1] - o2[1]);
        d[s] = 0;
        minPq.offer(new int[]{s, d[s]});
        // iterations all vertex
        while (!minPq.isEmpty()) {
            // chosen vertex u
            int[] uInfo = minPq.poll();
            int u = uInfo[0];
            int du = uInfo[1];
            if (du > d[u]) continue; // this vertex was already relaxations

            // relaxations vertex u
            List<int[]> adjs = graph[u];
            for (int[] adj : adjs) {
                int v = adj[0];
                int weight = adj[1]; // len of (u -> v)
                if (d[v] > d[u] + weight) {
                    // successful relaxation
                    d[v] = d[u] + weight;
                    p[v] = u;
                    minPq.offer(new int[]{v, d[v]});
                }
            }
        }
    }
}

注意，我们没有使用到暴力实现中的 标记数组，因为 if (du > d[u]) continue; 语句将会过滤所有已经执行过松弛操作的顶点 ，因此也就不需要标记数组了；同时只有成功放松边 后才会将顶点 （元素 {v, d[v]} ）加入优先队列中，这会过滤所有不可达的顶点。
